Preguntas

1) ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Es una buena práctica porque JavaScript permite validar en tiempo real y dar retroalimentación inmediata al usuario. Esto mejora mucho la experiencia de uso, reduce errores de captura y evita viajes innecesarios al servidor cuando los datos claramente no cumplen con el formato esperado.

2) ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Las validaciones en JavaScript pueden saltarse deshabilitando JavaScript en el navegador, modificando el DOM o el código desde DevTools, alterando variables en tiempo de ejecución, o enviando peticiones HTTP directas con herramientas externas sin usar el formulario. Por eso, nunca deben considerarse una barrera de seguridad definitiva.

3) Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque su objetivo principal es la eficiencia y la UX del lado cliente, no la seguridad final del sistema. Validar en frontend ayuda a guiar al usuario y a reducir carga innecesaria en el servidor. Pero, toda validación crítica y de seguridad debe repetirse y reforzarse en el servidor, que es el entorno controlado y confiable para tomar decisiones de aceptación o rechazo de datos.

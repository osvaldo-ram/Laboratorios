Preguntas y respuestas

1-.¿Cuál es la diferencia entre Internet y la World Wide Web?
El internet es una infraestructura global, funciona con protocolos TCP/IP para la transmisión de datos entre dispositivos y ofrece demasiados servicios. WWW es un servidor dentro de internet y se basa en el protocolo HTTP y hace uso de hipervínculos para conectar recursos. Y permite entrar a sitios web con navegadores como Chrome, Edge, Firefox, etc.
2.-¿Cuáles son las partes de un URL?
Se compone a partir de un protocolo (http/https), el dominio que identifica al servidor, la ruta donde se encuentra el recurso, los parámetros que sirven para enviar datos y son opcionales, y una parte que apunta a una parte específica de la página si es el caso.
3.-¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
Su propósito es hacer acciones que se quieren hacer a los recursos dentro de un servidor. GET se utiliza para obtener información sin modificarla. Head es parecido a GET pero solo devuelve los encabezados sin el contenido. POST permite enviar datos para crear un nuevo recurso. PUT se usa para reemplazar totalmente un recurso que ya existe, PATCH actualiza solo una parte del recurso y DELETE se encarga de eliminarlo.
4.-¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
POST, porque sirve para enviar información en el cuerpo de la solicitud y no en la URL, es útil por su seguridad cuando se trata de datos sensibles. Y esta diseñado para enviar datos al servidor y que este los procese, como un registro de información.
5.-¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
GET, porque la solicitud que se hace es para obtener el contenido de la página web sin modificar nada del servidor.
6.-Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
El código 200 significa "OK", por lo que no es un error, sino que, indica que la solicitud se procesó correctamente y el servidor devolvió el recurso solicitado.
7.-¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
Sí, porque el error 404 indica que el recurso solicitado no fue encontrado en el servidor. Esto puede ser por una mala configuración de la rutas, archivos eliminados, errores en la estructura del sitio etc.
8.-¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
Sí, porque el error 500 indica un fallo interno del servidor y no un problema del usuario. Esto puede ser por errores dentro del código, en la base de datos o en la lógica de la aplicación.
9.-¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
Que un elemento esté depreciado significa que ya no se recomienda su uso porque fue reemplazado por mejores prácticas, es decir, nuevas etiquetas en HTML 5 o prácticas basadas en CSS. Algunos ejemplos son front, center,b, i, frame y frameset.
10.-¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
HTML5 es una versión más moderna y completa que HTML4. Incluye nuevas etiquetas para organizar mejor las páginas, permite usar audio y video sin programas externos, mejora los formularios y funciona mejor en celulares. Deja el diseño a CSS y elimina etiquetas antiguas
11.-¿Qué componentes de estructura y estilo tiene una tabla?
En la estructura se usan etiquetas como "table" para crear la tabla "tr" para las filas, "th" para los encabezados y "td" para las celdas de datos. En el estilo se puede usar CSS para bordes, colores, tamaño de letra, alineación, espaciado y ancho de columnas.
12.-¿Cuáles son los principales controles de una forma HTML5?
"input" para capturar datos ya sea texto, correos, contraseñas, etc. "textarea" para escribir textos largos, "select" para listas desplegables con opciones."checkbox" "radio" para seleccionar una o varias opciones.
13.-¿Qué tanto soporte HTML5 tiene el navegador que utilizas?
Yo uso de navegador Microsoft Edge, y tiene muy buen soporte de HTML5, porque usa el mismo motor de Google Chrome. Por lo que todas las funciones como etiquetas semánticas, reproducción nativa de audio y video, APIs de almacenamiento local, son compatibles con mi navegador.
14.-¿Cuál es el ciclo de vida de los sistemas de información?
Son las etapas por las que pasa un sistema desde que surge la necesidad hasta que se deja de usar. Estas etapas son análisis, diseño, pruebas, implementación y mantenimiento.
15.-¿Cuál es el ciclo de desarrollo de sistemas de información?
Es un proceso técnico en fases para construir un sistema. Sus fases son análisis donde se identifican los requerimientos, de diseño donde se planea la solución, de desarrollo donde se programa el sistema, de pruebas para validar su funcionamiento, de implementación cuando se pone en operación, y de mantenimiento, para corregir errores y hacer mejoras durante su uso.
